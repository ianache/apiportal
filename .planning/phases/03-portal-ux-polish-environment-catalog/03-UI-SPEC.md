## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline chip padding, status-badge horizontal padding |
| sm | 8px | Compact element spacing (`gap-2`), button icon-to-label gap |
| md | 16px | Default element padding (`px-4`, `py-4`), card internal padding |
| lg | 24px | Section padding (`p-6`), sidebar group top spacing |
| xl | 32px | Header margin-bottom (`mb-8`), major in-page sections |
| 2xl | 48px | Page-level top padding (`p-8` = 32px ÔÇö exception noted below) |
| 3xl | 64px | Reserved for future full-page layout gaps |

Exceptions:
- Page content padding uses `p-8` (32px), not 48px ÔÇö consistent with existing `Projects.vue` and `ProjectDetail.vue` patterns.
- Sidebar sub-item indent: `pl-[28px]` (28px). 28 is a multiple of 4 and is a layout-derived value ÔÇö it aligns sub-item text with the text column of the parent nav row (parent row: `pl-4` icon padding + 20px icon + `gap-2` gap = ~28px total offset). This value was previously specified as `pl-3` (12px), which was incorrect; corrected to `pl-[28px]`.
- Touch targets for interactive icon-only buttons: minimum 32px ├ù 32px (existing shell pattern), not 44px (desktop-only application, touch target exception applies).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label | 12px (`text-xs`) | 700 (bold) | 1.25 |
| Heading | 24px (`text-2xl`) | 700 (bold) | 1.2 |
| Display | 48px (`text-5xl`) | 700 (bold) | 1.1 |

Weights in use: 400 (regular) and 700 (bold) ÔÇö exactly 2 weights.

Notes:
- Label role is used for uppercase tracking-widest section headers (e.g. "Versions", "Lifecycle Status", sidebar group labels, column headers, field labels). All uppercase labels use `tracking-widest`.
- Body role covers card descriptions, table cell text, modal body copy, tooltip text.
- Heading role covers modal titles (`text-2xl font-bold`), version detail section titles, card titles (`text-base font-bold` ÔÇö treated as compact heading).
- Display role is reserved for page-level `<h1>` elements (e.g. "APIs" on the catalog page, "Environments" on the environment catalog page). All display `<h1>` elements use `font-bold` (700).
- Font: Inter for all roles. `font-family: 'Inter', sans-serif` declared inline on root containers (existing pattern).
- Code/monospace: 12px (`text-xs`), `font-mono`, used for Version ID and SemVer values only. Collapses into the Label scale slot ÔÇö no separate size declaration needed.

### 3. Inline Edit ÔÇö API Name & Description (ProjectDetail.vue)

**Primary visual anchor:** The API name `<h1>` is the primary focal point of the ProjectDetail view. It sits at the top of the main content area, uses Display typography (48px `font-bold`), and is the first element a user interacts with when reviewing or editing an API.

**Editable fields:** `<h1>` (name) and `<p>` (description) in the header section.

**Edit eligibility (`canEditMetadata`):**
- `currentVersion?.status !== 'PUBLISHED'` AND user role is `API_DESIGNER` or `API_MANAGER`.

**Edit trigger:** Click on the field text when `canEditMetadata` is true. A `edit` icon (Material Symbols, 16px, `#a0a7b5`) appears on hover to signal editability.

**Name field edit mode:** `<input>` replacing `<h1>`, same font size (`text-5xl font-bold`), `border-b-2 border-[#0058bc]`, transparent background, full width. Saves on `blur` or `Enter`. Cancels on `Escape`.

**Description field edit mode:** `<textarea>` replacing `<p>`, `text-lg`, `border border-[#e3e2e7] rounded-xl px-3 py-2`, background `#f4f3f8`. Saves on `blur`. Cancels on `Escape`.

**Locked state (PUBLISHED or insufficient role):** Hover shows `lock` icon (`#a0a7b5`, 16px) with tooltip text "Locked ÔÇö published APIs cannot be edited".

**Optimistic UI:** Update local value immediately on save trigger. Revert to original value if the PATCH request fails. Show no intermediate loading indicator (save is fast).

**Error feedback:** On PATCH failure, restore original value and show inline error message below the field: "Could not save changes. Please try again." in `text-xs color: #991b1b`.

### 4. Version Status Filter & Sort (ProjectDetail.vue ÔÇö versions sidebar)

**Filter control:** `<select>` element in the sidebar header row, left of the "+ New" button.

**Options:** `ALL | DESIGN | REVIEW | APPROVED | PUBLISHED | DEPRECATED | RETIRED`

**Default selection:** `PUBLISHED` on initial load.

**Select styling:**
- `text-xs font-semibold`, `px-2 py-1.5`, `rounded-lg`.
- Background: `#f4f3f8`, border: `#e3e2e7`, text: `#414755`.
- Focus ring: `outline: 2px solid #0058bc`.
- Height matches the "+ New" button (32px effective height).

**Sort:** Always descending by `createdAt` (newest first). No sort toggle control needed.

**Deselection guard:** When the active `selectedVersion` is filtered out by a status change, auto-select the first item in `filteredVersions`. If `filteredVersions` is empty, set `selectedVersion` to `null` and show the empty state within the version detail panel.

**Empty filter state (version detail panel):** When `selectedVersion` is `null` because all versions are filtered out, display inside the main panel area: "No versions match the selected filter." (body text, `#414755`, centered vertically).

### 5. New Version Creation (ProjectDetail.vue modal)

**Trigger:** "+ New" button in versions sidebar header (existing button, needs handler wired).

**Modal (already built ÔÇö wire up):**
- SemVer input: placeholder "e.g. 1.0.0", validated as non-empty before enabling Create.
- Create button: disabled when input is empty or while loading. Loading state: spinner icon (`progress_activity animate-spin`) + label "CreatingÔÇª".
- On success: close modal, clear input, auto-select newly created version.
- On error: display error message inside modal below input: "Could not create version. [server message]" in `text-xs color: #991b1b`.

**RBAC guard:** Create button is only rendered when `canCreate` (Designer or Manager role). Non-privileged users do not see the "+ New" button.

### 6. Environment Catalog Page (`/settings/environments`)

**Layout:** Full-page content inside `Shell`. Matches the `p-8 max-w-7xl mx-auto` pattern from Projects.vue.

**Page header:**
- Eyebrow label: "Settings" (`text-xs font-bold uppercase tracking-widest`, `#0058bc`).
- `<h1>`: "Environments" (`text-5xl font-bold`, `#1a1b1f`).
- Subtitle: "Manage deployment environments and their tags." (`text-sm`, `#414755`).
- Primary CTA button (top-right): "Add Environment" (`#0058bc` background, white text).

**Table:**
- Columns: Slug | Name | Tags | Actions.
- Slug cell: monospace `text-sm font-bold`, `#1a1b1f`.
- Name cell: `text-sm`, `#414755`.
- Tags cell: inline chip list. Each tag: `text-xs font-semibold px-2 py-0.5 rounded-full`, background `#eff4ff`, text `#0058bc`. Max visible: 4, then "+N more" pill.
- Inline "Add tag" per row: clicking the `+` icon on a row opens a small inline input within the tags cell. Save on Enter or blur.
- Actions column: "Edit" icon button (`edit`, `#717786`, `aria-label="Edit environment"`) and "Delete" icon button (`delete`, `#991b1b`, `aria-label="Delete environment"`).

**Empty state:**
- Icon: `dns` (64px, `#c7c6d1`).
- Heading: "No environments yet".
- Body: "Add your first environment to start mapping API endpoints."
- CTA: "Add Environment" button (`#0058bc`).

**Create/Edit Environment modal:**
- Fields: Slug (text input, uppercase enforced, unique), Name (text input).
- Tags: free-form tag input (type + Enter to add, `├ù` to remove).
- Primary CTA: "Save Environment".
- Destructive CTA: "Delete Environment" (only in edit mode) ÔÇö triggers confirmation.

**Delete confirmation:** Inline within the modal, not a separate dialog. Replace button area with: "Delete this environment? This cannot be undone." + "Confirm Delete" (`#991b1b` background) + "Cancel" (outline).

### 7. Endpoint Panel in ProjectDetail (version detail section)

**Placement:** New "Endpoints" card below the existing "Implementation Details" and "Designer Insights" cards in the version detail panel.

**Card header:** "Endpoints" (`text-xs font-bold uppercase tracking-widest`, `#717786`) + "Register Endpoint" button (icon `add`, `#eff4ff` background, `#0058bc` text, `text-xs`). Button hidden when version status is `RETIRED`.

**Endpoint list (per registered environment):**
- Row: Environment slug badge (monospace, `#0058bc`) + Environment name + Base URL (`text-sm font-mono`, `#1a1b1f`) + delete icon button (`delete`, `#717786`, `aria-label="Delete endpoint"`).
- URL is rendered as plain text (not a link) to prevent accidental navigation.

**Empty state (no endpoints):** "No endpoints registered." (`text-sm italic`, `#717786`).

**Register Endpoint modal:**
- Dropdown: select Environment from catalog (slug ÔÇö name format).
- Text input: Base URL (placeholder "https://api.example.com/payments/v1").
- Validation: URL must start with `http://` or `https://`. Error: "Enter a valid URL starting with https://".
- Primary CTA: "Register Endpoint".
- On success: close modal, append endpoint to local list.
- On duplicate (same version + environment): server returns 409 ÔÇö show "An endpoint for this environment is already registered." in `text-xs color: #991b1b`.
