  <files>

    apps/bff/src/routes/environments.ts,

  <action>
1. **shared-types**: Add `Environment` (id, slug, name, tags) and `APIEndpoint` (id, versionId, environmentId, environment, baseUrl) interfaces. Add optional `endpoints` array to `APIVersion`.
2. **schema.prisma**: Add `Environment` model (slug unique) and `APIEndpoint` model (version-environment unique). Add relations and cascade deletes. Run `npx prisma migrate dev --name env-catalog` and `npx prisma generate`.
3. **environments.ts**: Create CRUD routes (`GET /environments`, `POST /environments`, `PATCH /environments/:id`, `DELETE /environments/:id`) with RBAC for `API_MANAGER` on mutating routes.
4. **apis.ts**: Add `POST /apis/:id/versions/:version/endpoints` (with environmentId, baseUrl body) and `DELETE /apis/:id/versions/:version/endpoints/:endpointId`. Update the existing `GET /apis/:id` route to include `endpoints: { include: { environment: true } }` in the nested `versions` fetch.
5. **app.ts**: Register the new `environmentRoutes`.
  </action>
  <verify>
    <automated>cd apps/bff && npx prisma validate && npm run test -- --run</automated>
  </verify>
  <done>Database and BFF correctly support Environment and APIEndpoint CRUD.</done>
