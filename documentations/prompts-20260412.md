# Resumen de Mejoras - Diseñador de Modelos E/R (12 de Abril de 2026)

Este documento resume las peticiones y mejoras implementadas en el Modelador de Entidad-Relación de Nexus API Manager durante esta sesión.

### 1. Arquitectura y Configuración
*   **Análisis inicial:** Creación de `GEMINI.md` para establecer las reglas, stack tecnológico y mandatos del proyecto.
*   **Publicación:** Gestión del flujo de Git para subir todos los avances a GitHub.

### 2. Mejoras Visuales y de UI
*   **Identificadores (Badges):** Implementación de etiquetas dinámicas "PK" (naranja) y "FK" (púrpura) en el panel de propiedades.
*   **Auto-ajuste de tablas:** Corrección del error de desbordamiento de texto permitiendo que las tablas ajusten su ancho dinámicamente según el contenido (especialmente útil para nombres de campo largos como `documento_identidad`).
*   **Optimización de espacio:** Reubicación del botón "+ Add Column" como un icono sutil alineado con la etiqueta de la sección.
*   **Indicadores de Zoom:** Visualización del porcentaje de escala actual en la barra de herramientas.

### 3. Precisión en el Trazado de Relaciones
*   **Conexión campo a campo:** Las líneas ahora nacen y mueren exactamente en la fila del campo correspondiente, proporcionando una visualización técnica precisa.
*   **Puntos de contacto:** Adición de círculos pequeños en los bordes de la tabla para delimitar el inicio y fin de la relación.
*   **Superposición (Z-Index):** Ajuste estructural del DOM para que las líneas se dibujen siempre por encima de las tablas, garantizando la visibilidad del punto de origen.

### 4. Interactividad y Navegación
*   **Panning (Desplazamiento):** Capacidad de mover el lienzo completo manteniendo presionada la tecla `Ctrl` y arrastrando el ratón.
*   **Zoom Dinámico:** Sistema de acercamiento y alejamiento con `Ctrl + Rueda del ratón`, con un rango de escala de 0.2x a 3x.
*   **Snap to Grid:** Funcionalidad para activar/desactivar el alineamiento automático a una rejilla de 20px, facilitando el orden visual del diagrama.
*   **Selección de Contexto:** 
    *   Clic en línea de relación -> Panel "Relationship Properties".
    *   Clic en área vacía del canvas -> Panel "Model Properties".

### 5. Atributos de Datos y Modelado
*   **Metadatos de Documentación:** Incorporación de campos de descripción para Tablas, Campos y el propio Canvas.
*   **Especificaciones Técnicas:** Adición de atributos de longitud (`length`) y lugares decimales (`decimalPlaces`) para tipos VARCHAR y DECIMAL.
*   **Automatización de FKs:** Creación automática de campos Foreign Key al arrastrar y soltar un campo (PK) sobre la zona "Drop to link" de otra tabla.
*   **Gestión Global:** Botón para eliminar una tabla del modelo global y de todas sus representaciones visuales.

### 6. Inteligencia Artificial (Contexto)
*   **Aislamiento de contexto en AI SQL:** La generación de DDL ahora es "consciente del canvas". Solo se procesan las tablas y relaciones visibles en la pestaña activa. Las llaves foráneas a tablas fuera del canvas se tratan automáticamente como atributos simples para asegurar que el script SQL generado sea válido y autocontenido.
